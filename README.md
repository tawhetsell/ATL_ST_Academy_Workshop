# Workshop Overview 

This repository contains the files related to a network science workshop held at the Georgia Institute of Technology 2026 Atlanta Science and Innovation Policy Academy. The workshop presents network science techniques for the analysis of international research collaboration data derived from Artificial Intelligence research from 2003 to 2023. The workshop covers data formatting, network visualization, and four types of inferential network models: ERGM, TERGM, BTERGM, and RSIENA.

# Network and Covariate Data

International research collaboration data are generated by parsing and disambiguating Artificial Intelligence tagged records from the Web of Science XML database for the years 2003, 2008, 2013, 2018, and 2023. The data are transformed into symmetrical weighted matrices from the raw XML data using Python scripts. Exogenous covariates are gathered from the Varieties of Democracy project and the World Bank Indicators. The data selection is intended to be illustrative of the general approach to network science on research collaboration data and not a description of a publication-ready work product.

# Repository File Description 

The data are imported, formatted, visualized, and analyzed using `ATL_ST_Workshop_RScript.R`. The R script is reproduced in `ATL_ST_Workshop_QMD.html` as a rendered Quarto HTML document for readability outside the R environment. `ATL_ST_Workshop_Slides.html` is a slide deck summarizing the approach. The folder `country_covariate_data` contains CSV files for the exogenous covariates. The folder `IRC_network_data` contains the parsed and disambiguated country-country matrices for each year of data.

## Reproducing the Harmonized IRC Matrices from WoS XML

Matrices contained in `IRC_network_data` were produced from Web of Science XML files using the Python scripts in the folder `python_scripts`.

First, place the raw Web of Science XML folders in your workspace. These files are proprietary and hence not provided here. For the workshop, the expected folder names are:

- `2003_CORE/`
- `2008_CORE/`
- `2013_CORE/`
- `2018_CORE/`
- `2023_CORE/`

Run the scripts in this exact order:

1. `python_scripts/subset_wos_ai_records.py`
2. `python_scripts/parse_ai_country_matrices.py`
3. `python_scripts/apply_country_crosswalk.py`
4. `python_scripts/harmonize_country_matrices.py`

These CLI commands can be used to run the scripts:

```bash
python3 python_scripts/subset_wos_ai_records.py 2003_CORE 2008_CORE 2013_CORE 2018_CORE 2023_CORE --output-dir subset_ai_xml --summary-csv subset_ai_xml/subset_summary.csv

python3 python_scripts/parse_ai_country_matrices.py subset_ai_xml --output-dir ai_country_raw

python3 python_scripts/apply_country_crosswalk.py ai_country_raw --crosswalk python_scripts/country_crosswalk.csv --output-dir ai_country_clean

python3 python_scripts/harmonize_country_matrices.py ai_country_clean --output-dir IRC_network_data
```

The file `python_scripts/country_crosswalk.csv` is not run directly, but is required by step 3. It standardizes country names into ISO3 country codes. Each script creates the input needed by the next script.
